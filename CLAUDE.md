## What This Is

A Kubernetes **Validating Admission Webhook** in Go that rejects `Deployment` and `StatefulSet` resources whose containers or initContainers are missing CPU/memory requests or limits. Enforcement only applies to namespaces labeled `resource-validation=enabled` (filtered by the `ValidatingWebhookConfiguration`, not Go code).

## Commands

All Make targets auto-detect whether you're inside the dev container (`/.dockerenv`) and delegate accordingly. The `_`-prefixed targets are the real implementations that run inside the container.

```bash
make dev              # build dev image and start with Air hot-reload (host only)
make shell            # open a shell in the running dev container
make test             # run unit tests (delegates to: go test ./... -v)
make kind-up          # full cluster setup: create kind cluster, generate TLS certs, build/load image, apply manifests
make test-integration # apply test fixtures and verify allow/deny behavior against the cluster
make test-clean       # remove integration test resources
make clean            # stop containers, delete kind cluster, remove certs/
```

Run a single test from inside the container:
```bash
go test ./webhook/... -run TestMissingLimits -v
```

**Request flow:** Kubernetes API server â†’ `POST /validate` â†’ `Handle()` â†’ `evaluate()` â†’ decode Deployment or StatefulSet â†’ `validatePodSpec()` â†’ collect violations â†’ allow or deny with message listing all violations.

**Validation:** All four fields must be non-zero for every container and initContainer: `resources.requests.cpu`, `resources.requests.memory`, `resources.limits.cpu`, `resources.limits.memory`.

## Key Design Decisions

- **No controller-runtime.** Only `k8s.io/api` and `k8s.io/apimachinery` â€” keeps the binary small and the dependency surface auditable.
- **`failurePolicy: Fail`** â€” if the webhook is unreachable, K8s blocks the admission request. Downtime in labeled namespaces will block all Deployment/StatefulSet creates and updates.
- **TLS certs are generated by `make kind-up`** using openssl with CN = `webhook-service.webhook-system.svc`. The `caBundle` placeholder `REPLACE_CA_BUNDLE` in `manifests/validatingwebhookconfiguration.yaml` is substituted at deploy time via `sed`.
- **Dev environment** is Docker Compose + Air (hot reload). Source is mounted at `/app` inside the container. The dev container also has access to `/var/run/docker.sock` for `kind` cluster management.
- **Env vars** configure the server: `PORT` (default `8443`), `HEALTH_PORT` (default `8080`), `TLS_CERT`, `TLS_KEY`.
