# K8s Resource Validation Admission Webhook — Implementation Plan

## Goal

A Kubernetes **Validating Admission Webhook** written in Go that rejects any `Deployment` or
`StatefulSet` whose containers or initContainers are missing CPU/memory resource requests or limits.
The webhook only enforces namespaces labeled `resource-validation=enabled`.

---

## Choices

| Parameter | Value |
|---|---|
| Language | Go 1.22 |
| Scope | `Deployment`, `StatefulSet` (CREATE + UPDATE) |
| Containers checked | `containers` + `initContainers` |
| Fields enforced | `resources.requests.cpu`, `resources.requests.memory`, `resources.limits.cpu`, `resources.limits.memory` |
| Namespace filter | label `resource-validation=enabled` |
| Failure policy | `Fail` (deny when webhook is unreachable) |
| Dev environment | Docker Compose + Air (hot reload) |
| TLS | Self-signed cert generated by `cert-gen` container / Makefile target |

## Implementation Steps

### Step 1 — Go module

```bash
go mod init github.com/yourorg/k8s-resource-webhook
```

Dependencies:
- `k8s.io/api` — Deployment, StatefulSet, AdmissionReview types
- `k8s.io/apimachinery` — runtime codec, resource.Quantity

No controller-runtime or heavy frameworks — keeps the binary small and auditable.

---

### Step 2 — Validation logic (`webhook/validator.go`)

```
validatePodSpec(name string, spec corev1.PodSpec) []string
```

Algorithm:
1. Merge `spec.Containers` + `spec.InitContainers` into one slice
2. For each container check all four fields:
   - `resources.requests["cpu"]`
   - `resources.requests["memory"]`
   - `resources.limits["cpu"]`
   - `resources.limits["memory"]`
3. Collect violations as human-readable strings:
   `"container \"api\": missing resources.limits.cpu"`
4. Return the violation list (empty = valid)

---

### Step 3 — Admission handler (`webhook/handler.go`)

```
POST /validate
```

Flow:
1. Read + JSON-decode `AdmissionReview` request
2. Switch on `request.resource.resource`:
   - `"deployments"` → decode `apps/v1.Deployment`, extract `.Spec.Template.Spec`
   - `"statefulsets"` → decode `apps/v1.StatefulSet`, extract `.Spec.Template.Spec`
3. Call `validatePodSpec`
4. Build `AdmissionResponse`:
   - No violations → `allowed: true`
   - Violations found → `allowed: false`, `status.message` lists all violations
5. JSON-encode and write `AdmissionReview` response

---

### Step 4 — HTTP server (`main.go`)

- `POST /validate` → admission handler
- `GET /healthz` → liveness/readiness probe (returns 200 OK)
- TLS loaded from env vars `TLS_CERT` / `TLS_KEY` (paths to mounted cert files)
- Listen address from env `PORT` (default `8443`), health from `HEALTH_PORT` (default `8080`)
- Graceful shutdown on SIGTERM

---

### Step 5 — TLS certificate generation

**Local dev** (`make certs`):
```
openssl req -x509 -newkey rsa:4096 \
  -keyout certs/tls.key -out certs/tls.crt \
  -days 365 -nodes \
  -subj "/CN=webhook" \
  -addext "subjectAltName=DNS:webhook,DNS:localhost,IP:127.0.0.1"
```

**Cluster deploy** (`make certs-cluster`):
- Same openssl command but CN = `webhook-service.webhook-system.svc`
- Outputs base64 for `manifests/secret.yaml` and `caBundle` in `ValidatingWebhookConfiguration`

---

### Step 6 — Kubernetes manifests

#### `validatingwebhookconfiguration.yaml`

```yaml
webhooks:
  - name: resource-validation.example.com
    namespaceSelector:
      matchLabels:
        resource-validation: "enabled"
    rules:
      - apiGroups: ["apps"]
        apiVersions: ["v1"]
        resources: ["deployments", "statefulsets"]
        operations: ["CREATE", "UPDATE"]
        scope: "Namespaced"
    clientConfig:
      service:
        name: webhook-service
        namespace: webhook-system
        path: /validate
      caBundle: <BASE64_CA>
    admissionReviewVersions: ["v1"]
    sideEffects: None
    failurePolicy: Fail
    timeoutSeconds: 10
```

#### `deployment.yaml`

- 2 replicas (HA)
- Resource requests + limits set (self-dogfooding)
- Liveness probe: `GET /healthz`
- Readiness probe: `GET /healthz`
- TLS cert mounted from Secret

---

### Step 7 — Production Dockerfile

Multi-stage build:
1. `FROM golang:1.22-alpine AS builder` — `CGO_ENABLED=0 go build`
2. `FROM gcr.io/distroless/static` — copy binary only

Final image is ~10MB with no shell.

## Validation Behavior Table

| Scenario | Decision |
|---|---|
| Container missing `resources` block entirely | Deny |
| Container has `requests` but no `limits` | Deny |
| Container has `limits` but no `requests` | Deny |
| Container missing only `memory` limit | Deny |
| initContainer missing CPU request | Deny |
| All containers + initContainers fully specified | Allow |
| Namespace without `resource-validation=enabled` | Skip (Allow) |
| Webhook server unreachable | Deny (`failurePolicy: Fail`) |

## Dev Workflow

```bash
# 1. Start dev environment (generates certs, starts webhook with hot reload)
docker compose up

# 2. Webhook is live at https://localhost:8443/validate

# 3. Edit any .go file → air reloads automatically

# 4. Run unit tests inside container
docker compose exec webhook go test ./...

# 5. Build production image
make build

# 6. Generate cluster certs and render manifests
make certs-cluster

# 7. Deploy to cluster
make deploy
```

## Testing Plan

### Unit tests (`webhook/validator_test.go`)
- All four fields present → no violations
- Missing one field → violation with correct container name and field
- Multiple containers, one bad → correct container named in violation
- initContainer violations caught

### Integration test (manual)
```bash
# Allow: all resources set
kubectl apply -f test/good-deployment.yaml

# Deny: missing limits
kubectl apply -f test/bad-deployment-no-limits.yaml

# Deny: missing requests
kubectl apply -f test/bad-statefulset-no-requests.yaml

# Skip: namespace without label
kubectl apply -f test/unlabeled-namespace-deployment.yaml
```
